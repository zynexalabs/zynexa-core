# Security Policy

## Reporting Vulnerabilities

**Do not open public issues for security vulnerabilities.**

If you discover a security vulnerability in Zynexa, please email security@zynexa.io with:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will acknowledge your report within 48 hours and work with you on a fix timeline.

## Security Best Practices for Users

- **Private Keys**: Never share your recovery phrase or private keys
- **Passwords**: Use strong, unique passwords
- **Backups**: Keep encrypted backups of important keys
- **Updates**: Always keep Zynexa updated to the latest version
- **Verification**: Verify authenticity of Zynexa before installation

## Zero-Knowledge Architecture

Zynexa is built with privacy-first principles:
- No server-side storage of private keys
- End-to-end encryption for all communications
- Zero-knowledge proofs for identity verification
- No metadata collection or tracking

## Compliance

- All code undergoes security review before release
- We maintain audit logs for infrastructure
- Regular security audits and penetration testing
- OWASP Top 10 compliance
